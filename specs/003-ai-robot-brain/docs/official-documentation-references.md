## Best Practices for Source-Grounding

### Verification Process
1. Always cite the most recent version of official documentation
2. Validate technical claims through independent testing
3. Cross-reference with academic literature when available
4. Include version numbers for software and documentation references
5. Link to specific sections or pages when referencing documentation

### Citation Format (APA)
For documentation: Author/Organization. (Year). Title (Version X.X). Source. URL
For academic papers: Author, A. A., & Author, B. B. (Year). Title of work. Publisher/Source, URL if available

### Quality Assurance
- All technical claims must be verifiable through official documentation
- Academic references should be from peer-reviewed sources
- Implementation examples should be tested and validated
- References should be current (preferably within last 5 years for rapidly evolving fields)
